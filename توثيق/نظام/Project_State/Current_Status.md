# سجل حالة مشروع نظام إدارة المحتوى الأكاديمي الذكي (S-ACM)

## تاريخ التحديث: 2026-01-09

## 1. نظرة عامة على المشروع

مشروع **نظام إدارة المحتوى الأكاديمي الذكي (S-ACM)** هو نظام يهدف إلى توفير منصة رقمية متكاملة لإدارة المحتوى الأكاديمي في جامعة العلوم والتكنولوجيا - فرع الحديدة. يجمع النظام بين وظائف رفع وتنظيم المواد الدراسية، نظام إشعارات ذكي، ووظائف الذكاء الاصطناعي (تلخيص المحاضرات وتوليد الأسئلة)، بالإضافة إلى نظام صلاحيات متقدم.

## 2. تحليل وثيقة المشروع الأولية

تم تحليل وثيقة المشروع الأصلية (ملف PDF بعنوان `_الفصول_الاربعه_مشروع_ادارة_المحتوئ_الاكاديمي_الذكي2.pdf`) بشكل شامل ودقيق. شمل التحليل:
- **الفصول الأربعة:** فهم معمق للمقدمة، الخلفية النظرية، تحليل النظام، والتصميم والتنفيذ.
- **المخططات:** تحليل جميع مخططات UML (ERD, Class, Use Case, Sequence, Activity) لفهم بنية النظام وسلوكياته.
- **واجهات المستخدم:** تحليل الـ 22 واجهة مستخدم المقترحة لفهم تجربة المستخدم المطلوبة.
- **النتائج:** تم إعداد تقرير تحليلي مفصل (Analysis.md) يلخص نقاط القوة ويقدم توصيات للتحسين.

## 3. خطط تنفيذ المشروع

بناءً على طلب المستخدم، تم إعداد ثلاث وثائق خطة رئيسية:
- **المسار الكامل للمشروع (Project_Roadmap.md):** يوضح المراحل الرئيسية للتطوير من الإعداد إلى التسليم.
- **خطة الـ 15 يوم (15_Day_Plan.md):** خطة تنفيذية مفصلة مقسمة يوم بيوم.
- **توزيع المهام (Team_Roles.md):** يحدد مسؤوليات كل عضو في الفريق (فريق التطوير وفريق الجودة والتوثيق).

## 4. القرارات التقنية الرئيسية

تم اتخاذ القرارات التقنية التالية بناءً على مناقشات مع المستخدم:

| الجانب | القرار المتخذ | السبب/الملاحظات |
|---|---|---|
| **بيئة التطوير** | Python Virtual Environment (venv) | بيئة افتراضية بسيطة ومباشرة لعزل مكتبات المشروع. |
| **IDE** | VS Code (موصى به) | خفيف، سريع، وتكامل ممتاز مع Python و Git. |
| **لغة البرمجة** | Python | اللغة الأساسية لإطار عمل Django. |
| **إطار العمل** | Django | إطار عمل ويب قوي وموثوق به. |
| **قاعدة البيانات** | PostgreSQL | قاعدة بيانات علائقية قوية ومناسبة لبيئات الإنتاج. |
| **الذكاء الاصطناعي** | Google Gemini API (2.5) | تم توفير مفتاح API من قبل المستخدم. |
| **تخزين الملفات** | سحابي (Telegram/OneDrive) | لضمان قابلية التوسع والأمان (سيتم تحديد الخيار الأفضل لاحقاً). |
| **إدارة الكود** | Git/GitHub | لتعاون الفريق، تتبع الإصدارات، وإدارة المهام (GitHub Issues). |
| **توثيق الفريق الثاني** | Microsoft Word | لإنشاء الفصول النهائية ودليل المستخدم. |

## 5. حالة مستودع GitHub (`Documentation`)

- تم إنشاء مستودع GitHub باسم `Documentation`.
- تم تطبيق نظام إصدارات للملفات، حيث يتم الاحتفاظ بنسخ مؤرخة (مثلاً `Analysis_v1.0_20260108_1839.md`) ونسخ "أحدث إصدار" (Latest) بدون تاريخ (مثلاً `Analysis.md`) لسهولة الوصول.
- يحتوي المستودع حالياً على:
    - وثيقة المشروع الأصلية (PDF).
    - تقرير التحليل الشامل (Analysis.md).
    - وثائق الخطط (Project_Roadmap.md, 15_Day_Plan.md, Team_Roles.md).
    - أدلة العمل التفصيلية للفريقين (Team1_Developers_Guide.md, Team2_QADoc_Guide.md).

## 6. الخطوة التالية المقترحة

الآن، الفريق جاهز للبدء في تنفيذ المشروع. الخطوة التالية هي أن يبدأ فريق التطوير (معين ومهند) في إعداد بيئة Python ومشروع Django الأولي كما هو موضح في `Guides/SETUP.md`.
