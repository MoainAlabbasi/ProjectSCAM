# المسار الكامل لمشروع نظام إدارة المحتوى الأكاديمي الذكي (S-ACM)

## مقدمة

هذا المستند يوضح الخطة الشاملة والمراحل الكاملة لتطوير مشروع S-ACM من البداية إلى النهاية. تم تصميم هذا المسار ليكون مرجعاً أساسياً للفريق، حيث يقسم العمل إلى مراحل منطقية قابلة للإدارة والتنفيذ.

---

### المرحلة 0: الإعداد والتجهيز (Setup & Preparation)

**الهدف:** تجهيز بيئة العمل والأدوات اللازمة لضمان بداية سلسة ومنظمة للمشروع.

| المهمة | الوصف التفصيلي |
|---|---|
| **1. إعداد بيئة التطوير الموحدة** | تثبيت البرامج الأساسية على أجهزة المبرمجين (أنت ومهند) لضمان تطابق البيئة وتجنب المشاكل. تشمل: Python 3.11+, Django, PostgreSQL, Git, و VS Code مع الإضافات الموصى بها (Python, Django, Prettier). |
| **2. تنظيم العمل على GitHub** | - **استراتيجية الفروع (Branching Strategy):** اعتماد استراتيجية GitFlow المبسطة: `master` للنسخة المستقرة، `develop` للتطوير المستمر، و `feature/feature-name` لكل ميزة جديدة.<br>- **إدارة المهام (Issues):** إنشاء `Issues` لكل مهمة في الخطة على مستودع GitHub وتعيينها للمبرمج المسؤول. |
| **3. إنشاء هيكل مشروع Django** | - إنشاء مشروع Django جديد (`django-admin startproject acm_project`).<br>- إنشاء تطبيق Django أساسي (`python manage.py startapp core`).<br>- ربط المشروع بقاعدة بيانات PostgreSQL.<br>- ضبط إعدادات المشروع (`settings.py`) مثل المنطقة الزمنية والملفات الثابتة. |

---

### المرحلة 1: بناء الأساس (Building the Foundation)

**الهدف:** بناء الهيكل الأساسي للنظام، ويشمل نماذج البيانات، نظام المستخدمين، والصلاحيات.

| المهمة | الوصف التفصيلي |
|---|---|
| **1. بناء نماذج البيانات (Models)** | ترجمة مخطط ERD الموجود في وثيقة التحليل إلى نماذج Django (`models.py`). يشمل ذلك إنشاء جميع الجداول (Users, Roles, Courses, Lectures, etc.) وتحديد العلاقات بينها (ForeignKey, ManyToManyField). |
| **2. إعداد لوحة تحكم المدير (Admin Panel)** | تسجيل النماذج التي تم إنشاؤها في `admin.py` لتوفير واجهة إدارية بسيطة تسمح بإضافة وتعديل البيانات مباشرة من لوحة تحكم Django. هذه الخطوة حيوية للاختبار وإدارة البيانات في المراحل الأولى. |
| **3. نظام إدارة المستخدمين والصلاحيات (RBAC)** | - **المصادقة (Authentication):** بناء وظائف تسجيل الدخول، تسجيل الخروج، وإنشاء حساب جديد.<br>- **التفويض (Authorization):** بناء نظام الأدوار والصلاحيات (Admin, Doctor, Student) للتحكم في الوصول إلى الصفحات والوظائف المختلفة. |

---

### المرحلة 2: تطوير الوظائف الأساسية (Core Features Development)

**الهدف:** برمجة الوظائف الرئيسية التي تخدم المستخدمين الأساسيين (الطالب والمدرس).

| المهمة | الوصف التفصيلي |
|---|---|
| **1. نظام إدارة المقررات والمحتوى** | - **واجهات المدرس:** بناء الصفحات التي تمكن المدرس من رفع الملفات (PDF, DOCX, etc.) وتصنيفها ضمن المقررات المخصصة له.<br>- **واجهات الطالب:** بناء الصفحات التي تمكن الطالب من تصفح المقررات المسجل بها وتنزيل المواد الدراسية. |
| **2. تصميم الواجهات الأمامية (Frontend)** | تحويل تصميمات الواجهات (UI) الموجودة في الوثيقة إلى قوالب Django (`.html`) باستخدام Bootstrap 5 و HTMX لتحقيق التجاوب والتفاعلية. |
| **3. ربط الواجهات بالمنطق البرمجي (Backend)** | كتابة الـ `views.py` والـ `urls.py` لربط الواجهات الأمامية بالوظائف البرمجية في الخلفية، مثل عرض البيانات من قاعدة البيانات ومعالجة النماذج (Forms). |

---

### المرحلة 3: تطوير الوظائف المتقدمة (Advanced Features Development)

**الهدف:** إضافة الميزات التي تعزز من قيمة النظام وتجعله ذكياً وتفاعلياً.

| المهمة | الوصف التفصيلي |
|---|---|
| **1. نظام الإشعارات (Notifications)** | بناء نظام يسمح للمدرسين بإرسال إشعارات للطلاب، وتظهر هذه الإشعارات في لوحة تحكم الطالب. |
| **2. لوحات التحكم والإحصائيات (Dashboards)** | تطوير لوحات التحكم الخاصة بالمسؤول والمدرس لعرض الإحصائيات والتقارير كما هو مصمم في الوثيقة. |
| **3. دمج وظائف الذكاء الاصطناعي (AI Integration)** | - **اختيار API:** تحديد واجهة برمجة تطبيقات (API) مناسبة مثل **OpenAI's GPT** أو **Gemini**.<br>- **التكامل:** بناء وظائف في Django تقوم بإرسال نصوص المحاضرات إلى الـ API واستقبال الملخصات أو الأسئلة وعرضها للمستخدم.<br>- **إدارة الاستخدام:** تطبيق نظام لتحديد عدد الطلبات المسموح بها لكل مستخدم (Rate Limiting). |

---

### المرحلة 4: الاختبار والنشر (Testing & Deployment)

**الهدف:** التأكد من جودة النظام وخلوه من الأخطاء، ثم نشره ليكون متاحاً للاستخدام.

| المهمة | الوصف التفصيلي |
|---|---|
| **1. الاختبار الشامل (Testing)** | - **اختبار الوحدة (Unit Testing):** كتابة اختبارات للتحقق من أن كل دالة تعمل كما هو متوقع.<br>- **اختبار التكامل (Integration Testing):** اختبار تفاعل المكونات مع بعضها (مثل رفع ملف وظهوره للطالب).<br>- **اختبار قبول المستخدم (UAT):** جعل أعضاء الفريق الآخرين (رياض ومحمد) والمشرف يجربون النظام ويسجلون أي ملاحظات أو أخطاء. |
| **2. التهيئة للنشر (Deployment Prep)** | - **إعدادات الإنتاج:** تعديل `settings.py` لتناسب بيئة الإنتاج (مثل إخفاء مفاتيح الـ API وتغيير إعدادات قاعدة البيانات).<br>- **إعداد ملفات البيئة:** تجهيز ملفات `.env` و `requirements.txt` للنشر. |
| **3. النشر على خدمة سحابية (Deployment)** | نشر المشروع على خدمة سحابية مجانية أو منخفضة التكلفة مثل **Railway** أو **Render** أو **PythonAnywhere** لجعله متاحاً عبر الإنترنت. |

---

### المرحلة 5: التوثيق والتسليم (Documentation & Handover)

**الهدف:** إعداد الوثائق النهائية للمشروع وتسليمه.

| المهمة | الوصف التفصيلي |
|---|---|
| **1. تحديث التوثيق النهائي** | تحديث وثيقة المشروع (الفصول 5 و 6) بنتائج الاختبار، صور من النظام النهائي، والتوصيات المستقبلية. |
| **2. إعداد دليل المستخدم** | كتابة دليل بسيط يشرح كيفية استخدام النظام لكل من الطالب والمدرس والمدير. |
| **3. تجهيز العرض التقديمي** | إعداد شرائح العرض التقديمي لمناقشة المشروع، مع التركيز على المشكلة، الحل، البنية التقنية، وعرض حي للنظام. |
